N-Queens Problem

A common problem presented for intelligent agents is the 8/N queens problem. In this problem an agent attempts to place queens on a chessboard without them being able to capture each other. For an empty chessboard the most number of queens that can be placed on the board is 8, one for each row or column. An alternate problem involving barriers modifies the maximum number thus creating the N-Queens problem.

Methods involving backtracking are typically used to solve this problem; however, for a short term project I solved this problem using an genetic algorithm. The aim of this project was to place as many queens on the board but I considered the maximum heuristic to be 9. The reason for this was part of how I decided to place my queens. I placed one queen for ever column for the board which would allow my agent to ignore vertical collisions.

From there it was the task of using a genetic algorithm to place the queens on the board and check for collisions. For each generation I set the score to 9 and subtracted from that score for each collision that occured. For the crossover I would first reduce the population by at most half based on a survival threshold. Basically, in order for a member of the current generation to reproduce they would need to get a score above the survival threshold. If they did then they would be crossbred for the next generation.

Additionally I also randomly mixed the population once they were selected to breed. This was done in hopes of avoiding local maximums and ensure that the desired heuristic would be met. While there was a few complications I eventually was able to tweak the agent to reach the desired heuristic. I do acknowledge that it may be possible to achieve a higher score if placing the queens on each row (possibly 11 in this case) and its something I want to consider attempting in the future.
